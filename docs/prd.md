# PRD — 입찰 공고 분석기

## 1. 배경 및 문제

선엔지니어링 수주전략팀은 나라장터 입찰 공고를 정기적으로 검토한다.
공고문은 분량이 많고 형식이 매번 달라 마감일·참가자격·제출서류를 하나씩 찾아 읽는 데 시간이 소요된다.
여러 공고를 동시에 검토할 때 비교가 어렵고, 제출서류는 매번 처음부터 작성해야 한다.

## 2. 목표

PDF 공고문을 업로드하면 핵심 정보가 자동으로 카드로 정리되고,
제출서류 준비 현황 관리와 실적·경력 문서 작성까지 한 화면에서 처리할 수 있는 웹 도구를 제공한다.

## 3. 사용자

- **주 사용자**: 선엔지니어링 수주전략팀 (비개발자 포함)
- 로그인·회원가입 없음
- 별도 설치 없이 브라우저에서 바로 사용

## 4. 사용자 스토리

| # | 스토리 |
|---|--------|
| U1 | 공고 PDF를 업로드하면 핵심 항목이 카드로 정리된다 |
| U2 | 마감일까지 며칠 남았는지 D-Day로 한눈에 파악한다 |
| U3 | 제출서류 체크리스트로 준비 진행률(%)을 관리한다 |
| U4 | 여러 공고를 나란히 놓고 비교한다 |
| U5 | 유사용역 실적·기술자 경력을 폼으로 입력해 .docx로 내보낸다 |
| U6 | 브라우저를 닫았다 열어도 이전 공고 목록이 그대로 남아있다 |

## 5. 기능 명세

### 5-1. PDF 업로드 및 텍스트 추출

- 드래그&드롭 또는 파일 선택으로 PDF 업로드
- `pdf.js`(pdfjs-dist)로 클라이언트 사이드에서 텍스트 추출
- 추출된 원문 텍스트를 키워드 패턴(정규식)으로 파싱해 아래 항목을 구조화
  - 공고명
  - 발주기관
  - 예정금액
  - 용역기간
  - 입찰 마감일
  - 참가자격 (업종·면허·실적 요건)
  - 제출서류 목록
- **"원문 확인 필요" 경고 배너**를 카드 상단에 항상 표시 (추출 정확도 보장 불가 명시)
- 추출 실패 항목은 "확인 필요" 텍스트로 표시하고 수동 편집 가능

### 5-2. ERP 대시보드 뷰 (메인 화면)

메인 화면은 ERP 스타일 대시보드로 구성한다.

**레이아웃 구조**
- 좌측 아이콘 사이드바: 대시보드 / 공고 목록 / 비교 / 내보내기 탐색
- 상단 바: 앱 제목 + "원문 확인 필요" 경고 배지 + PDF 업로드 버튼
- 콘텐츠 영역은 세 구역으로 구성

**① KPI 카드 행 (4개)**
- 전체 공고 수
- D-7 이내 마감 임박 건수 (빨간색 강조)
- 체크리스트 미완료 건수 (주황색 강조)
- 관리 중 공고의 평균 예정금액

**② 공고 목록 테이블 뷰**
- 업로드된 공고마다 한 줄(row)로 표시
- 좌측 컬러 바로 긴박도 표시: 빨강(D-7 이내) / 주황(D-8~14) / 없음(D-15+) / 회색(마감)
- 열 구성: 공고명+발주기관 / 예정금액 / 체크리스트 진행률 바 / D-Day 뱃지 / 상세 이동 버튼
- 마감 임박 공고가 상단에 정렬됨
- 마감된 공고는 흐리게(opacity 낮춤) 표시
- 행 클릭 시 상세 화면으로 이동

**③ 하단 2분할 패널**
- 좌측: 선택된 공고의 체크리스트 미리보기 (항목별 완료/미완료)
- 우측: 공고 비교 선택 패널 (선택된 공고 태그 표시 + 비교 보기 버튼)

**D-Day 뱃지 색상 규칙**
- 7일 이내: 빨간색 (`D-3` 등)
- 8~14일: 주황색
- 15일 이상: 초록색
- 마감 경과: 회색 (`마감`)

### 5-3. 제출서류 체크리스트

- 추출된 제출서류 목록을 체크박스 리스트로 표시
- 항목별 체크 시 진행률(%) 실시간 업데이트
- 진행률 프로그레스 바 표시 (0~100%)
- 항목 추가·삭제·수정 가능 (추출 누락 대비)
- 체크 상태는 localStorage에 자동 저장

### 5-4. 다중 공고 비교 뷰

- 2개 이상 공고 선택 후 "비교하기" 버튼 클릭
- 선택한 공고를 가로로 나란히 배치한 비교 테이블 표시
- 비교 항목: 공고명·발주기관·예정금액·용역기간·마감일·참가자격·제출서류 수·진행률
- 항목 간 차이가 있는 셀 강조 표시

### 5-5. 실적·경력 폼 및 .docx 내보내기

- 공고 상세 화면 내 탭으로 제공
- **유사용역 실적 폼**
  - 용역명 / 발주기관 / 계약금액 / 용역기간 / 수행내용 (행 추가 가능)
- **기술자 경력 폼**
  - 성명 / 직위 / 자격증 / 투입기간 / 담당업무 (행 추가 가능)
- `.docx` 내보내기 버튼 클릭 시 `docx` 라이브러리로 Word 문서 생성 후 다운로드
- 문서 상단에 공고명·발주기관 자동 삽입

### 5-6. 데이터 영속성 (localStorage)

- 저장 항목: 공고 목록 전체(추출 데이터 + 체크 상태 + 폼 입력값)
- 업로드·편집·체크 변경 시 자동 저장 (디바운스 500ms)
- 공고 삭제 시 해당 데이터만 제거
- 저장 용량 초과(5MB 초과 추정) 시 사용자에게 경고

## 6. 비기능 요구사항

| 항목 | 내용 |
|------|------|
| 인증 | 로그인·회원가입 없음 |
| 데이터 저장 | 클라이언트 localStorage만 사용, 서버 전송 없음 |
| AI 분석 | 미포함 (키워드 파싱만 사용) |
| 경고 표시 | "원문 확인 필요" 배너 항상 노출, 숨김 불가 |
| 반응형 | 데스크톱 우선 (최소 1280px), 모바일 대응은 선택 |
| 브라우저 지원 | Chrome 최신 버전 기준 |
| 접근성 | 키보드 탐색 기본 지원 |
